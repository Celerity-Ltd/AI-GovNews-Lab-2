Step 2 — Build the policy rewriter

A ReAct agent that takes the original document plus the gap report and rewrites the document to close every gap. Uses the predeployed web_search tool to ground its wording in the real ISO 9001:2015 standard.

Step 2.1 — Setup

Open the Build area first
From any page in watsonx Orchestrate, click the hamburger menu (☰) in the top-left and choose Build — that’s where you create and manage agents.
Agent name[Your initials] policy-rewriter
DescriptionRewrites a quality management document to address every gap in an ISO 9001:2015 gap analysis. Returns the full rewritten document with [UPDATED - ISO 9001:2015 clause X.X] markers and a Summary of Changes.
StyleReAct
Toolsweb_search (predeployed in this tenant)
Create an agent — Name + Description
From the Agent Builder home, click Create new agent → Create from scratch and fill in Name and Description.
Agent style — switch to ReAct
Below the agent details, switch Agent style to ReAct.

Step 2.2 — Attach the web_search tool

  1. Scroll down to the Toolset section.
  2. Click Add tool → Add from local instance.
  3. Search for web_search and click Add.
  4. Verify web_search appears in the tool list.
web_search returns title, source URL and snippet for each result — the agent uses these to ground its rewritten clauses.
Toolset — Add tool
Scroll to the Toolset section and click Add tool, then choose Add from local instance.

Step 2.3 — Behaviour

Paste into the agent's Behaviour field
# Role
Expert quality writer for ISO 9001:2015. Inputs: original document text + gap analysis. Output: fully rewritten document that closes every gap.

## Rules
- Keep original structure and numbering. Add new sections only where needed.
- Do not remove compliant content — only expand or improve it.
- For every NON-COMPLIANT or PARTIAL clause, rewrite or add the relevant section.
- Use the web_search tool to look up precise ISO 9001:2015 clause wording.
- Mark every new or changed section with [UPDATED - ISO 9001:2015 clause X.X] right after the heading.

## Output formatting (important — must be Word-friendly)
Return the rewritten document in clean markdown so it copies cleanly into Microsoft Word:
- Document title as a single # heading at the top.
- Each main section as a ## heading (e.g. ## 4. Context of the organisation).
- Each subsection as a ### heading (e.g. ### 4.1 Understanding the organisation).
- Body text in normal paragraphs separated by blank lines — no walls of text.
- Use **bold** for inline emphasis (defined terms, role names, key clauses).
- Use hyphen bullets (- ) for unordered lists and 1. 2. 3. for numbered lists.
- One blank line before and after every heading, list, and table.
- Do NOT wrap the document in a code block or triple backticks — output the markdown directly.
- After the document, append a final ## Summary of Changes section as a markdown table with columns: Section | Clause | What changed.

This format renders nicely in chat AND pastes into Word with headings, bold and lists preserved (use Word's 'Keep Source Formatting' on paste).
Behavior — paste the Instructions here
Scroll to the Behavior section and paste the block above into the Instructions field.
